# Project Templates

Templates for PROJECT.md and project-config.json generation.

## PROJECT.md Template

```markdown
# {Project Name}

## Vision

{1-2 sentence vision statement describing the big picture goal}

## Core Problem

{What specific problem this solves and for whom}

## Target Users

{Description of primary users and their context}

### User Needs

- {Need 1}
- {Need 2}

## Technical Constraints

- {Constraint 1 - platform, performance, security, deployment}
- {Constraint 2}

## Tech Stack

- **Language:** {language}
- **Frontend:** {framework or "N/A"}
- **Backend:** {framework or "N/A"}
- **Database:** {database or "TBD"}
- **Testing:** {test framework}

## Requirements

### Validated

(None yet — ship to validate)

### Active

- [ ] {Requirement 1}
- [ ] {Requirement 2}
- [ ] {Requirement 3}

### Out of Scope

- {Exclusion 1} — {why}
- {Exclusion 2} — {why}

## Key Decisions

| Decision | Rationale | Status |
| -------- | --------- | ------ |
| {Choice} | {Why} | Pending |

## Success Criteria

- [ ] {Measurable criterion 1}
- [ ] {Measurable criterion 2}
- [ ] {Measurable criterion 3}

---

*Generated by `/tiki:new-project` on {YYYY-MM-DD}*
*Edit this file directly to update project context.*
```

## project-config.json Structure

```json
{
  "createdAt": "ISO-8601 timestamp",
  "updatedAt": "ISO-8601 timestamp",
  "version": 1,
  "project": {
    "name": "string",
    "vision": "string",
    "coreProblem": "string"
  },
  "users": {
    "targetUsers": "string description",
    "needs": ["array", "of", "needs"]
  },
  "technical": {
    "constraints": ["array", "of", "constraints"],
    "techStack": {
      "language": "string",
      "frontend": "string or null",
      "backend": "string or null",
      "database": "string or null",
      "testing": "string or null"
    }
  },
  "scope": {
    "successCriteria": ["array", "of", "criteria"],
    "outOfScope": ["array", "of", "exclusions"]
  },
  "research": {
    "completed": false,
    "path": null
  }
}
```

## Section Guidelines

| Section | Purpose | Length |
| ------- | ------- | ------ |
| Vision | Elevator pitch | 1-2 sentences |
| Core Problem | Pain point addressed | 2-3 sentences |
| Target Users | Who benefits | 2-3 sentences |
| User Needs | Problems solved | 3-5 bullets |
| Constraints | Hard requirements | As needed |
| Tech Stack | Implementation choices | Structured list |
| Requirements | What to build | Organized by status |
| Success Criteria | Measurable outcomes | 3-5 checkboxes |
| Out of Scope | Explicit exclusions | 3-5 bullets |

## Minimal Template

For quick experiments or learning projects:

```markdown
# {Project Name}

## Goal

{What you're building or learning}

## Tech

- {Primary technology}

## Success

- [ ] {One clear outcome}
```
