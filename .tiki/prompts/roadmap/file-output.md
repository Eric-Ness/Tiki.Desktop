# ROADMAP.md File Output

Generate a ROADMAP.md file in the project root for persistent documentation.

## File Structure

```markdown
# Project Roadmap

Generated: 2026-01-18
Releases: 4 (2 shipped, 1 active, 1 planned)

## Overview

[ASCII timeline content]

## Release Details

### v1.0 - Foundation (Shipped 2026-01-10)
- 3 issues completed
- Requirements: EXEC-01, EXEC-02, QUAL-01

### v1.1 - Release Management (Active - 40% complete)
- 2/5 issues completed
- Requirements: MULTI-01, MULTI-02

### v1.2 - Visualization (Planned)
- 0/3 issues started
- Target requirements: VIZ-01, VIZ-02

## Requirements Traceability Matrix

[Requirements coverage table - only if requirements.json exists]

---
*Generated by Tiki `/tiki:roadmap --output`*
```

## Header Section

- **Generated**: Current date (YYYY-MM-DD)
- **Releases**: Total count with breakdown by status

## Overview Section

Include the full ASCII timeline output (see ascii-format.md).

## Release Details Section

For each release (sorted by version):

**Status strings:**
- Shipped: `Shipped {date}`
- Active: `Active - {progress}% complete`
- Planned: `Planned`

**Content:**
- Shipped: `{count} issues completed`
- Active: `{completed}/{total} issues completed`
- Planned: `0/{total} issues started`

**Requirements line:**
- Shipped/Active: `Requirements: {comma-separated IDs}`
- Planned: `Target requirements: {comma-separated IDs}`
- Omit line if no requirements linked

## Requirements Traceability Matrix

Only include if `.tiki/requirements.json` exists. Use requirements coverage table format.

## Success Message

After writing file, display:

```
ROADMAP.md generated successfully.

Location: ./ROADMAP.md
Releases: {count} ({status breakdown})
Issues: {total} total

View at: file://{project_path}/ROADMAP.md
```

## Generation Steps

1. Get current date for header
2. Count releases by status for header
3. Generate ASCII timeline for Overview
4. Build Release Details for each release
5. Add Requirements Traceability if requirements exist
6. Add footer with generation note
7. Write to ./ROADMAP.md using Write tool
8. Display success message
