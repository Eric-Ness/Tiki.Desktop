# GitHub Comment Template

Template for posting analysis decisions to GitHub issues.

## Comment Structure

Format the comment as follows:

```markdown
## Deep Dive Analysis

Analysis performed by `/tiki:release-review` to address concerns from pre-planning review.

### Concerns Addressed

#### 1. [Category]: {concern summary}

**Analysis:** {brief explanation of the concern}

**Decision:** {chosen approach}

**Rationale:** {why this approach was chosen}

**Code References:**
- `path/to/file.ts:42` - {what's relevant here}

---

#### 2. [Category]: {concern summary}

**Analysis:** {brief explanation}

**Decision:** {chosen approach}

**Rationale:** {why}

---

### Summary

| Concern | Category | Decision |
|---------|----------|----------|
| {brief} | {cat} | {approach} |
| {brief} | {cat} | {approach} |

### Recommendations

{Any additional notes for planning phase}

---
*Generated by `/tiki:release-review`. Proceed with `/tiki:plan-issue {number}` when ready.*
```

## Guidelines

- Keep each section concise but complete
- Include specific code references where relevant
- Focus on decisions made, not alternatives considered
- Use clear, actionable language
- Table provides quick reference for later review
